# Object Storage Daily Size Chart Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an admin Object Storage grouped bar chart that compares daily confirmed R2 size (red) to daily cleanup deletion size (green) for the selected period.

**Architecture:** Extend the existing `/v1/admin/object-storage` response with server-generated, UTC-zero-filled daily activity rows. A small local React chart component renders paired keyboard-focusable CSS bars with a shared byte scale, a date/value tooltip, and an explicit no-activity state; no charting package is added.

**Tech Stack:** Go 1.x, PostgreSQL/sqlc, Next.js/React/TypeScript, CSS custom properties, Node test runner, Playwright regression suite.

- [ ] **Step 3: Add the sqlc daily aggregation query and regenerate it.**

  Add this named query after `GetAdminObjectStoragePeriodCleanupRuns`:

  ```sql
  -- name: ListAdminObjectStorageDailyActivity :many
  WITH confirmed AS (
    SELECT
      (uploaded_at AT TIME ZONE 'UTC')::date AS day,
      COALESCE(SUM(size_bytes), 0)::bigint AS confirmed_bytes,
      0::bigint AS deleted_bytes
    FROM media
    WHERE status = 'uploaded'
      AND uploaded_at >= sqlc.arg(period_from)::timestamptz
      AND uploaded_at < sqlc.arg(period_to)::timestamptz
    GROUP BY 1
  ), deleted AS (
    SELECT
      (finished_at AT TIME ZONE 'UTC')::date AS day,
      0::bigint AS confirmed_bytes,
      COALESCE(SUM(deleted_bytes), 0)::bigint AS deleted_bytes
    FROM media_cleanup_runs
    WHERE finished_at >= sqlc.arg(period_from)::timestamptz
      AND finished_at < sqlc.arg(period_to)::timestamptz
    GROUP BY 1
  )
  SELECT
    day,
    SUM(confirmed_bytes)::bigint AS confirmed_bytes,
    SUM(deleted_bytes)::bigint AS deleted_bytes
  FROM (
    SELECT * FROM confirmed
    UNION ALL
    SELECT * FROM deleted
  ) activity
  GROUP BY day
  ORDER BY day ASC;
  ```

  Expected: `api/internal/db/media_cleanup_runs.sql.go` gains `ListAdminObjectStorageDailyActivity`, a `ListAdminObjectStorageDailyActivityParams` with `PeriodFrom`/`PeriodTo`, and a row type exposing `Day`, `ConfirmedBytes`, and `DeletedBytes`.

- [ ] **Step 4: Extend the handler store, response, and zero-fill mapper.**

  Add the generated query to `adminObjectStorageStore`, query it with the existing `from` and `to` timestamps, and include the mapped result in `adminObjectStorageResponse`.

  ```go
  type adminObjectStorageDailyActivityResponse struct {
      Date           string `json:"date"`
      ConfirmedBytes int64  `json:"confirmed_bytes"`
      DeletedBytes   int64  `json:"deleted_bytes"`
  }

  func adminObjectStorageDailyActivity(rows []db.ListAdminObjectStorageDailyActivityRow, from, to time.Time) []adminObjectStorageDailyActivityResponse {
      totals := make(map[string]db.ListAdminObjectStorageDailyActivityRow, len(rows))
      for _, row := range rows {
          totals[row.Day.Time.UTC().Format("2006-01-02")] = row
      }
      start := time.Date(from.UTC().Year(), from.UTC().Month(), from.UTC().Day(), 0, 0, 0, 0, time.UTC)
      end := time.Date(to.UTC().Year(), to.UTC().Month(), to.UTC().Day(), 0, 0, 0, 0, time.UTC)
      out := make([]adminObjectStorageDailyActivityResponse, 0, int(end.Sub(start).Hours()/24)+1)
      for day := start; !day.After(end); day = day.AddDate(0, 0, 1) {
          key := day.Format("2006-01-02")
          row := totals[key]
          out = append(out, adminObjectStorageDailyActivityResponse{Date: key, ConfirmedBytes: row.ConfirmedBytes, DeletedBytes: row.DeletedBytes})
      }
      return out
  }
  ```

  Preserve the existing `writeQueryError` path when the new query errors. Update the existing migration/query contract list to include `-- name: ListAdminObjectStorageDailyActivity :many`.

  Add this API model before `AdminObjectStorageResponse` and reference it from that response:

    date: string;
    confirmed_bytes: number;

  }

  Create `object-storage-daily-chart.tsx` with this public boundary:

  ```tsx
  "use client";

  import { useMemo, useState } from "react";
  import type { AdminObjectStorageDailyActivity } from "@/lib/api";
  import { fmtBytes } from "../_components/admin-ui";

  export function ObjectStorageDailyChart({ rows }: { rows: AdminObjectStorageDailyActivity[] }) {
    const [activeDate, setActiveDate] = useState<string | null>(null);
    const peak = useMemo(() => Math.max(1, ...rows.flatMap((row) => [row.confirmed_bytes, row.deleted_bytes])), [rows]);
    const active = rows.find((row) => row.date === activeDate) ?? null;
    if (rows.every((row) => row.confirmed_bytes === 0 && row.deleted_bytes === 0)) {
      return <div className="aos-chart-empty">No Confirm or DELETE activity in this period.</div>;
    }
    return (
      <div className="aos-chart" role="group" aria-label="Daily Confirm and DELETE size">
        <div className="aos-chart-legend"><span><i className="aos-chart-confirm" />Confirm size</span><span><i className="aos-chart-delete" />DELETE size</span></div>
        <div className="aos-chart-plot">
          {rows.map((row, index) => <DailyGroup key={row.date} row={row} maxBytes={peak} showLabel={index === 0 || index === rows.length - 1 || index % Math.ceil(rows.length / 7) === 0} active={activeDate === row.date} onActivate={setActiveDate} />)}
        </div>
        {active && <div className="aos-chart-tooltip" role="status">{active.date}: Confirm {fmtBytes(active.confirmed_bytes)} · DELETE {fmtBytes(active.deleted_bytes)}</div>}
      </div>
    );
  }
  ```

  `DailyGroup` renders two `<button>` bars with percent heights derived from `maxBytes`, `aria-label` containing date/value/color meaning, and `onMouseEnter`, `onFocus`, and `onBlur` handlers that update `activeDate`. Each button has a transparent focus ring that meets the page's existing keyboard styling.
